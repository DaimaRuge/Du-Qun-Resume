## 📚 核心知识库

### 1. OpenClaw 核心架构

**定义**：
OpenClaw 是一个自托管的多渠道 AI 智能体 Gateway 网关，支持 WhatsApp、Telegram、Discord、iMessage 等主流聊天平台。

**核心组件**：
```
┌─────────────────────────────────────────┐
│          OpenClaw Gateway               │
│  ┌───────────────────────────────────┐  │
│  │   Multi-channel Router           │  │
│  │   - WhatsApp                      │  │
│  │   - Telegram                      │  │
│  │   - Discord                       │  │
│  │   - iMessage                      │  │
│  │   - Plugin Channels (Mattermost) │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │   Session Manager                 │  │
│  │   - Per-sender isolation          │  │
│  │   - Multi-agent routing           │  │
│  │   - Context management            │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │   Agent Core (Pi)                 │  │
│  │   - Tool execution                │  │
│  │   - Memory & context              │  │
│  │   - Multi-model support           │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │   Media Handler                   │  │
│  │   - Image processing              │  │
│  │   - Audio/video                   │  │
│  │   - Document handling             │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
         │                    │
    ┌────▼────┐          ┌────▼────┐
    │  CLI    │          │ Web UI  │
    └─────────┘          └─────────┘
```

**关键特性**：
1. **自托管**：运行在你的硬件上，你的规则
2. **多渠道**：一个 Gateway 服务多个聊天平台
3. **智能体原生**：为编程智能体设计，支持工具使用、会话、记忆
4. **开源**：MIT 许可证，社区驱动

### 2. 核心概念

**Sessions（会话）**：
- **Per-sender isolation**：每个发送者独立的会话
- **Multi-agent routing**：按智能体、工作区或发送者隔离
- **Context management**：上下文管理和压缩
- **Session persistence**：会话持久化

**Agents（智能体）**：
- **Pi Agent**：默认的编程智能体
- **Tool execution**：工具调用和执行
- **Memory system**：记忆系统（MEMORY.md + daily notes）
- **Multi-model support**：支持多种 LLM 提供商

**Channels（渠道）**：
- **WhatsApp**：通过 Baileys 库实现
- **Telegram**：通过 Telegram Bot API
- **Discord**：通过 Discord.js
- **iMessage**：macOS 原生支持
- **Plugin channels**：可扩展插件系统

**Tools（工具）**：
- **File operations**：read, write, edit
- **Shell execution**：exec, process
- **Web browsing**：browser, web_search, web_fetch
- **Messaging**：message (multi-channel)
- **Image analysis**：image
- **Feishu integration**：feishu_doc, feishu_drive, feishu_wiki
- **Memory management**：memory_search, memory_get

**Gateway（网关）**：
- **Single source of truth**：会话、路由、渠道连接的唯一事实来源
- **REST API**：HTTP API 端点
- **WebSocket**：实时通信
- **Web Control UI**：浏览器控制界面

### 方向2：部署实践

**目标**：提供从开发到生产的完整部署方案

**部署场景**：

1. **本地开发环境**
   ```bash
   # 安装
   npm install -g openclaw@latest

   # 新手引导
   openclaw onboard --install-daemon

   # 启动 Gateway
   openclaw gateway --port 18789
   ```

2. **VPS/云服务器部署**
   - **操作系统**：Ubuntu 22.04+, CentOS 8+
   - **依赖**：Node.js 22+, npm
   - **进程管理**：systemd, PM2
   - **反向代理**：Nginx, Caddy
   - **SSL/TLS**：Let's Encrypt

3. **Docker 容器化**
   ```dockerfile
   FROM node:22-alpine
   WORKDIR /app
   RUN npm install -g openclaw@latest
   EXPOSE 18789
   CMD ["openclaw", "gateway", "--port", "18789"]
   ```

4. **Kubernetes 集群部署**
   - **Deployment**：无状态部署
   - **Service**：LoadBalancer/NodePort
   - **ConfigMap**：配置管理
   - **Secret**：凭证管理
   - **PersistentVolume**：数据持久化

5. **安全配置**
   - **Token 认证**：API token 管理
   - **Allowlists**：IP/用户白名单
   - **Rate limiting**：请求限流
   - **Encryption**：端到端加密
   - **Audit logging**：审计日志

6. **监控与日志**
   - **Prometheus**：指标收集
   - **Grafana**：可视化监控
   - **ELK Stack**：日志聚合
   - **Sentry**：错误追踪

**输出**：
- 部署脚本和配置文件
- 运维手册
- 监控仪表板
- 故障排查指南
